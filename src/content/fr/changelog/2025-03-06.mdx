<Badge variant="noLink" className="m-0">
  <span>Amélioration</span>
</Badge>

- Nous avons augmenté la limite du nombre de chemins pouvant être affichés dans les parcours utilisateurs de 100k à 500k.
- Lorsque la connexion échoue, nous affichons désormais un message convivial.
- Nous ignorons les caractères `|` lorsqu'ils sont utilisés dans les CSS : cela a permis d'éviter d'analyser certains éléments spécifiques pour certains clients.

<Badge variant="noLink" className="m-0">
  <span>Fixer</span>
</Badge>

- Corrigé : Seuls les utilisateurs ayant des droits d'administrateur pouvaient créer/modifier/supprimer des instantanés. Cette fonction est désormais disponible pour les utilisateurs disposant d'un accès complet.
- Faux positifs sur les clics de rage : Lorsque de nombreux défilements étaient effectués sur mobile, cela pouvait être considéré comme des clics de rage, comme nous l'avons vu sur le site web d'un client lorsque les utilisateurs faisaient défiler les pages de manière très agressive. Cependant, cela ne pose pas de problème sur mobile. Ce problème est corrigé depuis le 5 mars 2025 et n'est pas rétroactif. Cela explique également le nombre élevé de clics de rage.
