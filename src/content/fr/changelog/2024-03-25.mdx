## Plusieurs améliorations du tunnel :
- Le tunnel montre par défaut la rétention des utilisateurs à partir de l'étape précédente.
- Le tunnel affiche le nombre de visiteurs depuis la première étape du tunnel.
- L'ajout d'événements à un tunnel peut se faire en recherchant par nom d'événement ou nom de label.

## Amélioration du plugin Chrome (version 3.6.3) :
- Les classes Tailwind CSS sont désélectionnées par défaut dans le plugin Chrome.
- Le temps d'hésitation s'affiche comme N/A pour l'analyse mobile car c'est une métrique non mobile uniquement.

## Autres améliorations :
- Lors de la création d'un nouveau label, le nom du label devait être saisi deux fois.
