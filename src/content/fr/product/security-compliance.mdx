<BlockFeature title='FAQs'>
  <MdxAccordion title='Comment fonctionne Air360 ?'>Une fois que le pixel Air360 est installé sur votre site web, lorsqu'un nouveau visiteur y accède, toutes les pages visitées, les mouvements de souris, les clics ou les défilements sont capturés par défaut. Cependant, nous ne capturons aucun champ de saisie ni rien qui pourrait contenir des informations personnelles identifiables (PII). Air360 fournit également des API et des paramètres pour affiner ces réglages et s'assurer qu'aucune donnée sensible n'est jamais capturée.</MdxAccordion>
  <MdxAccordion title='Quelles données sont collectées ?'>
  Nous suivons automatiquement chaque interaction que vos utilisateurs peuvent avoir sur votre site.
  - Clics
  - Défilement
  - Survol de la souris
  - Vue de page
  - Informations de la couche de données (basées sur une liste blanche)

  Il n'y a qu'une seule exception. Nous n'enregistrerons jamais ce que l'utilisateur tape dans une zone de texte de votre site web ou application. Cela pour une raison très simple : nous ne pouvons jamais savoir comment est cette information.

Nous collectons également des informations à partir des navigateurs des utilisateurs telles que :
Langue du navigateur
- Plateforme
- Appareil
- Système d'exploitation
- Adresse IP (*)

(*) L'adresse IP est uniquement utilisée pour récupérer la géolocalisation de l'utilisateur, mais elle n'est pas stockée dans notre base de données.
</MdxAccordion>
<MdxAccordion title='Quels types de cookies sont utilisés ?'>Depuis octobre 2024, Air360 ne repose plus sur les cookies et est une solution sans cookies.</MdxAccordion>

Les données que nous collectons par défaut peuvent devenir des données personnelles lorsque vous vous connectez à nos API pour transmettre des identifiants d'utilisateur ou des données de propriété utilisateur.</MdxAccordion>
<MdxAccordion title='Air360 ralentit-il votre site web ou votre application ?'>
Une des questions que nous recevons souvent chez Air360 concerne la possibilité que le pixel Air360 ralentisse votre site web. La réponse courte est non.

Réponse longue : Le pixel de suivi Air360 est chargé de manière asynchrone et à la fin de la balise de fermeture &lt;/body&gt;. Cela signifie que notre script est techniquement incapable de bloquer le chargement correct de votre site web. De plus, le pixel de suivi Air360 est hébergé sur un CDN (Cache Delivery Network), ce qui rend la latence de chargement extrêmement faible. Grâce à notre CDN, le JS se charge généralement en moins de 30 ms. Le JavaScript d'Air360 a également une empreinte mémoire très faible, utilisant moins de 20KB (compressé) de bande passante. C'est beaucoup plus petit que la plupart des scripts. C'est également beaucoup, beaucoup plus petit qu'une image même de faible résolution. Toutes ces mesures garantissent que le pixel de suivi Air360 ne ralentira jamais l'expérience utilisateur lorsque vos utilisateurs naviguent sur votre site web. En plus de toutes les mesures mentionnées ci-dessus, nous continuons à surveiller et à optimiser la performance du pixel Air360.
</MdxAccordion>
